Ejercicio 1

Almacenar valores en registros (hex)

Almacene los siguientes valores en los registros indicados, usando notación hexadecimal:
a. AX = 00000000b   b. BX = 01010101b   c. CX = 00001111b   d. DX = 110011100b

Conversión a hexadecimal

RegistroBinarioHexadecimal
AX00000000b00h
BX01010101b55h
CX00001111b0Fh
DX110011100b019Ch

¿Cómo se convierte? Cada 4 bits equivalen a 1 dígito hexadecimal. Para DX, el número binario tiene 9 bits: lo completamos a 16 bits añadiendo ceros a la izquierda (0000000110011100b), luego agrupamos de 4 en 4: 0000 0001 1001 1100 = 019Ch.

Solución .MODEL SMALL .STACK .DATA .CODE inicio: mov ax, @data mov ds, ax mov ax, 00h ; AX = 00000000b mov bx, 55h ; BX = 01010101b mov cx, 0Fh ; CX = 00001111b mov dx, 019Ch ; DX = 110011100b mov ax, 4C00h int 21h END inicio
Ejercicio 2

Copiar CX a AX, BX y DX

Escriba un programa que copie el contenido del registro CX en los registros AX, BX y DX.

La instrucción mov destino, fuente copia el valor de fuente en destino sin modificar fuente. Así que podemos usar CX como origen tres veces.

Solución .MODEL SMALL .STACK .DATA .CODE inicio: mov ax, @data mov ds, ax ; Supongamos que CX ya tiene algún valor, ej. CX = 0042h mov ax, cx ; AX = CX mov bx, cx ; BX = CX mov dx, cx ; DX = CX → ahora AX=BX=DX=CX mov ax, 4C00h int 21h END inicio
Ejercicio 3

Sumar los cuatro registros de propósito general

Realice un programa que sume los contenidos de los cuatro registros de propósito general y guarde el resultado en AX.

En alto nivel sería: AX = AX + BX + CX + DX. En Assembler, la instrucción ADD destino, fuente suma y guarda en el destino. Vamos acumulando en AX:

Paso 1: AX = AX + BX. Paso 2: AX = AX + CX (que ahora tiene el valor original AX+BX). Paso 3: AX = AX + DX.

Solución inicio: mov ax, @data mov ds, ax ; Supongamos AX=10, BX=20, CX=30, DX=40 add ax, bx ; AX = AX + BX = 10+20 = 30 add ax, cx ; AX = AX + CX = 30+30 = 60 add ax, dx ; AX = AX + DX = 60+40 = 100 mov ax, 4C00h int 21h END inicio
Ejercicio 4

Calcular año de nacimiento

Dada la edad de una persona almacenada en AX y el año actual en CX, calcule y almacene en DX el año de nacimiento.

La fórmula es simple: año de nacimiento = año actual - edad. Primero copiamos CX a DX, y luego le restamos AX. Otra opción: copiar CX a DX, luego sub dx, ax.

Solución inicio: mov ax, @data mov ds, ax ; Supongamos AX = 25 (edad), CX = 2025 (año actual) mov dx, cx ; DX = año actual sub dx, ax ; DX = año - edad = año de nacimiento ; DX = 2025 - 25 = 2000 mov ax, 4C00h int 21h END inicio

Ejercicio 7

Operaciones con variables del ejercicio 6

Con las declaraciones del ejercicio anterior, indique cómo realizar las siguientes operaciones:

b. Copiar en AL el tercer elemento (7) del arreglo impares

a. Copiar primoGrande en AX

Ejercicio 12

Sumar arreglo de 20 bytes

Escriba un programa que sume los números de un arreglo de 20 elementos (byte) que comienza en la posición almacenada en BX, y guarde la suma en AL.
Solución .DATA arreglo DB 20 DUP (1) ; Ejemplo: 20 elementos con valor 1 .CODE inicio: mov ax, @data mov ds, ax mov bx, offset arreglo ; BX = dirección del arreglo mov cx, 20 ; 20 elementos mov al, 0 ; Acumulador = 0 mov si, 0 ; Índice = 0 sumar: add al, ds:[bx + si] ; AL += elemento actual inc si ; Siguiente byte loop sumar ; CX--, si CX≠0 repetir ; AL contiene la suma total mov ax, 4C00h int 21h END inicio

Usamos ds:[bx + si] (direccionamiento relativo): BX apunta al inicio del arreglo y SI se incrementa en cada iteración para movernos entre elementos. Como son bytes, SI avanza de 1 en 1.

Ejercicio 13

¿Qué cambia si el arreglo es de words?

Si los elementos fueran de tamaño word (2 bytes), ¿qué cambios deben hacerse?

Cambios necesarios:

  1. Declarar con DW en vez de DB
  2. Usar AX como acumulador (no AL, ya que los elementos son de 16 bits)
  3. Usar ADD AX, ds:[BX + SI]
  4. Incrementar SI de 2 en 2 (add si, 2) ya que cada word ocupa 2 bytes
Cambios clave arreglo DW 20 DUP (1) ; DW en vez de DB ... mov ax, 0 ; Acumulador de 16 bits sumar: add ax, ds:[bx + si] ; Sumar word completo add si, 2 ; Avanzar 2 bytes (no 1) loop sumar

Ejercicio 16

IEEE 754 — Extraer exponente e invertir signo

Dado un número IEEE 754 de precisión simple en nroReal, copie el exponente en la variable exponente e invierta el signo.
Datos .DATA nroReal DD -0.000065 ; En IEEE: B888509C ; Little endian: 9C 50 88 B8 exponente DB ?

Formato IEEE 754 simple (32 bits):

Bit 31 = signo, Bits 30-23 = exponente, Bits 22-0 = mantisa.

B888509Ch en binario = 1 01110001 00010000101000010011100

Signo = 1 (negativo), Exponente = 01110001b = 71h = 113

En memoria little endian: byte 0 = 9Ch, byte 1 = 50h, byte 2 = 88h, byte 3 = B8h

El exponente se extrae de los bytes 2 y 3. Byte 3 (B8h) tiene el signo en su bit 7 y los bits 30-24 en los bits 6-0. Byte 2 (88h) tiene el bit 23 del exponente en su bit 7.

Solución .DATA nroReal DD -0.000065 exponente DB ? .CODE inicio: mov ax, @data mov ds, ax ; --- Extraer exponente --- mov al, nroReal[3] ; AL = B8h (byte alto) and al, 7Fh ; Quitar bit de signo → 38h shl al, 1 ; Mover 7 bits arriba → 70h mov ah, nroReal[2] ; AH = 88h (byte bajo del exponente) shr ah, 7 ; Obtener bit 23 → 01h or al, ah ; Combinar → 71h = 113 mov exponente, al ; Guardar exponente ; --- Invertir signo --- mov al, nroReal[3] ; AL = B8h xor al, 80h ; Invertir bit 7 → 38h mov nroReal[3], al ; Guardar → ahora es positivo mov ax, 4C00h int 21h END inicio

exponente = 71h (113 decimal). El signo se invirtió: B888509C → 3888509C (de negativo a positivo).

Ejercicio 17

Convertir carácter ASCII a decimal

Si el carácter es un dígito válido ('0'-'9'), convertirlo a decimal y guardarlo en digito. Si no, guardar FFh.

En ASCII, el carácter '0' = 30h, '1' = 31h, ..., '9' = 39h. Para convertir un dígito ASCII a su valor decimal, simplemente le restamos '0' (30h). Por ejemplo: '7' - '0' = 37h - 30h = 7.

Solución .DATA character DB "7" digito DB ? .CODE inicio: mov ax, @data mov ds, ax mov al, character ; ¿Es menor que '0'? cmp al, '0' jb no_valido ; Si AL < '0', no es dígito ; ¿Es mayor que '9'? cmp al, '9' ja no_valido ; Si AL > '9', no es dígito ; Es un dígito válido: convertir sub al, '0' ; AL = valor decimal (ej: '7'-'0'=7) mov digito, al jmp fin no_valido: mov digito, 0FFh ; No es un dígito válido fin: mov ax, 4C00h int 21h END inicio
Ejercicio 18

Invertir el orden de los bits

Dada la variable var, almacene en inv el contenido con los bits invertidos: b7b6b5b4b3b2b1b0 → b0b1b2b3b4b5b6b7

No es lo mismo "invertir" (NOT, que cambia 0↔1) que "revertir" (ponerlos en orden inverso). El ejercicio pide revertir el orden de los 8 bits.

Estrategia: Usamos SHR para sacar el bit menos significativo de var (va al carry flag), y luego RCL para insertarlo en inv. Repetimos 8 veces. SHR saca bits por la derecha, RCL los pone por la izquierda de inv.

Solución .DATA var DB 10110100b ; B4h — ejemplo inv DB ? .CODE inicio: mov ax, @data mov ds, ax mov al, var ; AL = 10110100b mov cl, 8 ; 8 bits a procesar mov ah, 0 ; Resultado (acumulador) reversar: shr al, 1 ; Mover bit 0 de AL al carry rcl ah, 1 ; Mover carry al bit 0 de AH dec cl jnz reversar ; Repetir 8 veces mov inv, ah ; Guardar resultado ; inv = 00101101b = 2Dh (b0b1b2b3b4b5b6b7 de B4h) mov ax, 4C00h int 21h END inicio
Ejercicio 19

Debug de programa con subrutina

Ejecutar paso a paso el siguiente programa y responder las preguntas.
.model small .stack .data numeros DW 15, 22, 37, 46, 00h resultado DW ? .code inicio: mov ax, @data mov ds, ax mov ax, offset numeros push ax call subrutina pop ax mov resultado, cx mov ax, 4C00h int 21h subrutina proc push bp mov bp, sp mov bx, ss:[bp + 4] mov cx, 0 recorrer: mov dx, ds:[bx] cmp dx, 0 je fin add cx, dx add bx, 2 jmp recorrer fin: pop bp ret subrutina endp end inicio

a. ¿Dirección de memoria de numeros?

numeros es la primera variable en .DATA, por lo que su dirección de desplazamiento es 0000h dentro del segmento de datos. Su dirección absoluta depende del segmento asignado por el SO.

b. Estado de la pila después de PUSH BP

(De SP hacia arriba/abajo en la pila):

Dirección (relativa)Contenido
SP+0 (tope)Valor antiguo de BP
SP+2Dirección de retorno (IP del CALL)
SP+4offset numeros (0000h)

c. ¿Valor de BX luego de la segunda iteración?

Primera iteración: BX = offset (0000h), lee 15. BX += 2 → 0002h. Segunda iteración: BX = 0002h, lee 22. BX += 2 → 0004h. BX = 0004h.

d. ¿Dirección y valor final de resultado?

resultado está en el desplazamiento 000Ah (después de los 10 bytes de numeros: 5 words × 2 bytes). El valor final: CX = 15+22+37+46 = 120 (0078h).

e. ¿Qué función cumple la subrutina?

Suma todos los elementos del arreglo de words hasta encontrar un 0 (valor centinela). Devuelve la suma en CX. Es equivalente a: sumar todos los elementos no nulos de un arreglo terminado en 0.

Ejercicio 20

Subrutina: mayor de dos números

Escriba la subrutina mayor que recibe dos enteros de 16 bits en BX y CX, y devuelve en AX el mayor.
Solución mayor proc cmp bx, cx jge bx_es_mayor ; Si BX >= CX, saltar mov ax, cx ; CX es mayor → AX = CX jmp fin_mayor bx_es_mayor: mov ax, bx ; BX es mayor → AX = BX fin_mayor: ret mayor endp

Usamos CMP + JGE (Jump if Greater or Equal, con signo). Si BX ≥ CX, saltamos a bx_es_mayor y copiamos BX a AX. Si no, copiamos CX a AX. En ambos casos, AX queda con el mayor.

Ejercicio 21

Subrutina mayor recibiendo direcciones

Reescriba mayor recibiendo las direcciones de los valores en SI y DI.
Solución mayor proc mov ax, ds:[si] ; AX = primer valor cmp ax, ds:[di] ; Comparar con segundo valor jge fin_mayor ; Si AX >= segundo, AX ya es mayor mov ax, ds:[di] ; El segundo es mayor → AX = segundo fin_mayor: ret mayor endp

Ahora los valores no están en registros sino en memoria. Usamos direccionamiento indirecto: ds:[si] y ds:[di] acceden a las direcciones apuntadas por SI y DI.

Ejercicio 24

Contar bits en 1

Crear subrutinas para contar la cantidad de bits 1 en un byte (dato8) y en una word (dato16).

a. contarUnosByte: AL → CX

Subrutina contarUnosByte proc push bx mov cx, 0 ; Contador de unos mov bh, 8 ; 8 bits a revisar check_bit: shr al, 1 ; Mover bit 0 al carry jnc no_es_uno ; Si carry=0, el bit era 0 inc cx ; Era 1, incrementar contador no_es_uno: dec bh jnz check_bit pop bx ret contarUnosByte endp

Recorre cada bit de AL usando SHR (que pone el bit menos significativo en el carry). Si el carry está en 1, el bit era 1 y sumamos al contador.

b. contarUnosWord: AX → CX

Subrutina contarUnosWord proc push bx mov bl, al ; Guardar byte bajo en BL mov al, ah ; Procesar byte alto primero call contarUnosByte ; CX = unos del byte alto push cx ; Guardar parcial mov al, bl ; Procesar byte bajo call contarUnosByte ; CX = unos del byte bajo pop dx ; DX = unos del byte alto add cx, dx ; CX = total pop bx ret contarUnosWord endp

Escriba longString que reciba la dirección de un string terminado en 0 por la pila y devuelva su longitud en AX.
Subrutina longString proc push bp mov bp, sp push si mov si, ss:[bp + 4] ; Dirección del string mov ax, 0 ; Contador de longitud contar: cmp byte ptr ds:[si], 0 je fin_contar ; Si encontramos el 0, terminamos inc ax ; Incrementar longitud inc si ; Siguiente carácter jmp contar fin_contar: pop si pop bp ret 2 ; Limpiar 1 parámetro (2 bytes) longString endp

cmp byte ptr ds:[si], 0 compara el byte apuntado por SI con 0. El byte ptr le dice al ensamblador que estamos trabajando con un byte de memoria (no un word). Usamos ret 2 porque hay un solo parámetro de 16 bits (2 bytes) en la pila.

Ejercicio 28

Subrutina: contar palabras en un string

Escriba contarPalabras que reciba la dirección de un string terminado en 0 y devuelva en AX la cantidad de palabras.

Estrategia: Recorremos el string carácter por carácter. Usamos un flag (BL) para saber si estamos dentro de una palabra o no. Cuando encontramos un carácter que NO es espacio y el flag indica que no estábamos en una palabra, incrementamos el contador. Esto cuenta el inicio de cada palabra nueva.

Subrutina contarPalabras proc push bp mov bp, sp push si push bx mov si, ss:[bp + 4] ; Dirección del string mov ax, 0 ; Contador de palabras mov bl, 0 ; Flag: 0=separador, 1=en palabra loop_str: cmp byte ptr ds:[si], 0 je fin_palabras cmp byte ptr ds:[si], ' ' je es_espacio ; No es espacio: ¿estamos ya en una palabra? cmp bl, 1 je siguiente ; Ya contamos esta palabra inc ax ; Nueva palabra detectada mov bl, 1 ; Ahora estamos en una palabra jmp siguiente es_espacio: mov bl, 0 ; Ya no estamos en una palabra siguiente: inc si jmp loop_str fin_palabras: pop bx pop si pop bp ret 2 contarPalabras endp
Ejercicio 29

Concatenar dos strings

Dados str1 y str2, concatene str2 al final de str1 usando una subrutina que reciba las direcciones por la pila.

Estrategia: 1) Encontrar el final de str1 (buscar el 0). 2) Copiar str2 carácter por carácter a partir de ahí. Asegurarse de que str1 tenga espacio suficiente para la concatenación.

Subrutina concatenar proc push bp mov bp, sp push si push di push bx mov si, ss:[bp + 4] ; Dirección de str1 mov di, ss:[bp + 6] ; Dirección de str2 ; Paso 1: encontrar el final de str1 find_end: cmp byte ptr ds:[si], 0 je found_end inc si jmp find_end ; Paso 2: copiar str2 al final de str1 found_end: mov bl, ds:[di] ; Leer carácter de str2 mov ds:[si], bl ; Escribirlo al final de str1 cmp bl, 0 je done ; Si era el 0, terminamos inc si inc di jmp found_end done: pop bx pop di pop si pop bp ret 4 ; Limpiar 2 parámetros concatenar endp

Llamada

.DATA str1 DB "ALVAREZ", 0, 10 DUP (0) ; Espacio extra str2 DB "JULIAN", 0 .CODE push offset str2 ; 2do: dirección de str2 push offset str1 ; 1ro: dirección de str1 call concatenar ; str1 ahora = "ALVAREZJULIAN"

El truco está en declarar str1 con espacio de sobra (10 DUP (0)) para que la concatenación entre sin problemas. Si str1 mide 7 caracteres y str2 mide 6, necesitamos al menos 14 bytes (7+6+1 para el 0 final).
