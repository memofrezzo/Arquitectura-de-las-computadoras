Introducción

Assembler 8086

Aprendé a programar en lenguaje ensamblador desde cero, con explicaciones en español y ejercicios resueltos paso a paso.

¿Qué es Assembler?

Es un lenguaje de programación que está a un paso del lenguaje de máquina. Cada instrucción que escribís corresponde exactamente a una instrucción que ejecuta el microprocesador.

Pensalo así: si en Python escribís resultado = 5 + 2, por debajo el procesador está haciendo tres pasos separados. En Assembler los ves todos.

¿Para qué sirve aprenderlo?

Para entender qué pasa realmente cuando un programa corre. Las abstracciones de los lenguajes de alto nivel ocultan muchos detalles que Assembler revela.

También se usa en sistemas operativos, drivers, firmware embebido y situaciones donde necesitás control absoluto del hardware.

Los Registros de la CPU

Un registro es una pequeña área de memoria dentro del propio procesador. Son ultra-rápidos (no necesitan pasar por el bus del sistema como la RAM). En Assembler los usamos como si fueran variables, pero son limitados en cantidad.

Analogía: Si la RAM es un cuaderno, los registros son los dedos de tu mano. Tenés pocos, pero podés operar con ellos instantáneamente.

Instrucciones Básicas

El formato general de casi toda instrucción es: INSTRUCCIÓN destino, fuente. El resultado siempre queda en el primer operando (destino).

InstrucciónSintaxisEquivalente en alto nivelQué hace
MOV MOV dest, fuente dest = fuente Copia el valor de fuente en dest
ADD ADD dest, op2 dest = dest + op2 Suma op2 al dest y guarda en dest
SUB SUB dest, op2 dest = dest - op2 Resta op2 a dest y guarda en dest
INC INC dest dest = dest + 1 Incrementa en 1. Más rápido que ADD dest, 1
DEC DEC dest dest = dest - 1 Decrementa en 1. Más rápido que SUB dest, 1
XCHG XCHG op1, op2 swap(op1, op2) Intercambia los valores de ambos operandos
AND AND dest, op2 dest = dest & op2 Operación lógica AND bit a bit
OR OR dest, op2 dest = dest | op2 Operación lógica OR bit a bit
XOR XOR dest, op2 dest = dest ^ op2 OR exclusivo. XOR reg, reg → pone el reg en 0
SHL SHL dest, 1 dest = dest << 1 Corrimiento a izquierda. Equivale a ×2
SHR SHR dest, 1 dest = dest >> 1 Corrimiento a derecha. Equivale a ÷2 (unsigned)
MUL MUL op AX = AL × op (bytes)
DX:AX = AX × op (words)		 Multiplicación sin signo. Resultado en AX o DX:AX
DIV DIV op AL = AX / op

NEG NEG dest dest = -dest Complemento a 2 (cambia el signo)
Regla importante: No se puede hacer una operación entre dos direcciones de memoria. Si querés copiar memoria a memoria, primero cargás en un registro y de ahí movés. Y las constantes numéricas solo pueden ser el segundo operando (la fuente, no el destino).

Estructura de un Programa en Assembler

Todo programa en Assembler para MASM (Intel 8086) tiene un esqueleto estándar. Las líneas marcadas se repiten siempre igual — podés memorizarlas como "la fórmula".

programa.asm — esqueleto estándar
1 .MODEL SMALL ; modelo de memoria pequeño (código+datos caben en 64 KB c/u)
2 .STACK ; reserva espacio para la pila (stack)
3 .DATA ; ── aquí van las declaraciones de variables ──
4 resultado DW ? ; por ejemplo: variable "resultado" de 16 bits
5 .CODE ; ── aquí va nuestro código ──
6 inicio:
7 mov ax, @data ; inicializar el segmento de datos ←┐
8 mov ds, ax ; (siempre igual, copia @data en DS) ←┘
9 ; ─────── nuestro código va aquí ───────────────────
10 mov ax, 4C00h ; terminar el programa ←┐
11 int 21h ; (siempre igual) ←┘
12 END inicio ; marca el fin del archivo
Segmento de Datos (.DATA)

Donde declarás tus variables. Equivale al área de "declaración de variables" de cualquier lenguaje. El ensamblador reserva el espacio en memoria y asigna un nombre simbólico a cada dirección.

Segmento de Código (.CODE)

Donde va tu programa. Siempre arranca con las dos líneas de @data (para inicializar DS) y termina con las dos líneas de 4C00h / int 21h.

Ejercicio 1

Ejercicio 1 — Cargar valores binarios en registros

1
Asignar valores binarios usando notación hexadecimal
AX = 00000000b  |  BX = 01010101b  |  CX = 00001111b  |  DX = 11001100b

Paso 1: convertir de binario a hexadecimal

El truco es agrupar los bits de a 4, empezando desde la derecha. Cada grupo de 4 bits da un dígito hex.

RegistroBinarioGrupos de 4HexDecimal
AX 0000 0000 0000 | 0000 0000h0
BX 0101 0101 0101 | 0101 → 5|5 0055h85
CX 0000 1111 0000 | 1111 → 0|F 000Fh15
DX 1100 1100 1100 | 1100 → C|C 00CCh204

Paso 2: escribir el programa

ejercicio1.asm
1 .MODEL SMALL
2 .STACK
3 .DATA ; no hay variables en este ejercicio
4 .CODE
5 inicio:
6 mov ax, @data
7 mov ds, ax
8 mov ax, 0000h ; AX = 00000000b = 0
9 mov bx, 0055h ; BX = 01010101b = 55h = 85
10 mov cx, 000Fh ; CX = 00001111b = 0Fh = 15
11 mov dx, 00CCh ; DX = 11001100b = CCh = 204
12 mov ax, 4C00h
13 int 21h
14 END inicio
El enunciado dice "utilice notación hexadecimal (BCH)" — esto significa que debés asignar los valores usando su equivalente en hex, no directamente en binario. En Assembler podés escribir binario con b al final, pero la consigna pide el formato h.

2

→ AX = CX
2
mov bx, cx
Copia el valor de CX en BX.
→ BX = CX
3
mov dx, cx
Copia el valor de CX en DX.
→ DX = CX

Ejercicio 3

Ejercicio 3 — Sumar los cuatro registros de propósito general en AX

3
AX = AX + BX + CX + DX
ax = ax + bx + cx + dx

ADD siempre suma al primer operando. Vamos encadenando las sumas sobre AX:

1
add ax, bx
AX pasa a valer AX + BX
→ AX = AX + BX
2
add ax, cx
Al AX actual (que ya tiene AX+BX) le sumamos CX
→ AX = AX + BX + CX
3
add ax, dx
Sumamos DX al resultado acumulado
→ AX = AX + BX + CX + DX ✓
BX, CX y DX no se modifican. ADD solo cambia el primer operando (destino). Si querés verificarlo: después del paso 1, BX sigue siendo el mismo.
Ejercicio 4

Ejercicio 4 — Calcular el año de nacimiento

4
AX = edad  |  CX = año actual  |  DX = año en que nació
dx = cx - ax   (año_nacimiento = año_actual - edad)

No podemos hacer DX = CX - AX directamente en un paso. Primero copiamos CX en DX y luego le restamos AX:

ejercicio4.asm — código relevante
1 mov dx, cx ; DX = año actual (copiamos CX en DX)
2 sub dx, ax ; DX = DX - AX = año_actual - edad = año_nacimiento
¿Por qué no sub cx, ax? Porque el enunciado pide guardar el resultado en DX. Si hicieras sub cx, ax, modificarías CX y el resultado quedaría ahí, no en DX.

Variables en Assembler

Una variable en Assembler es simplemente un nombre simbólico para una dirección de memoria. El ensamblador se encarga de reemplazar ese nombre por la dirección real.

Las variables se declaran en la sección .DATA con este formato:

NOMBRE TIPO VALOR_INICIAL
TipoTamañoRango (sin signo)Ejemplo
DB1 byte (8 bits)0 – 255 (00h – FFh)edad DB 25
DW1 word (16 bits)0 – 65535 (0000h – FFFFh)saldo DW 0A00h
?sin inicializar—resultado DW ?
DUParreglo repetido—arr DB 10 DUP(0)

Arreglos, Caracteres y Strings

; Arreglo de bytes con valores
pares DB 2, 4, 6, 8

; Acceso a elementos
mov al, pares ; = pares[0] = 2
mov al, pares[1] ; pares[1] = 4
mov al, pares[2] ; pares[2] = 6
; Arreglo de words (16 bits)
numGrandes DW 1002h, 55A1h

; Acceso (desplaz. en BYTES)
mov ax, numGrandes ; [0] 1er elem
mov ax, numGrandes[2] ; [2] 2do elem
mov ax, numGrandes[4] ; [4] 3er elem
¡Atención con los arreglos de words! El índice entre corchetes siempre es en bytes, no en elementos. Si cada elemento ocupa 2 bytes (DW), el segundo elemento está en el índice 2, el tercero en 4, etc.
; String = arreglo de caracteres
nombre DB "Mariano"

; Arreglo sin inicializar (DUP con ?)
buffer DB 50 DUP(?)

; Arreglo inicializado en 0
zeros DB 50 DUP(0)
; Caracter ASCII: equivalentes
letra DB 41h ; 'A' en ASCII
letra DB 'A' ; equivalente
letra DB 65 ; equivalente

; Obtener dirección de una variable
mov bx, offset nombre
; BX ahora tiene la dirección
Ejercicio 6

Ejercicio 6 — Contenido del segmento de datos

6
¿Cómo quedan estas variables en memoria?
Declaraciones a analizar
numeroPrimo DB 7
primoGrande DW 9973
impares DB 1, 3, 7, 9
arreglo_5 DB 5 DUP(0Fh)
nombre DB "Luis"

Las variables quedan en memoria consecutivamente, en el mismo orden que las declaramos.

VariableDirección relativaBytes en memoria (hex)Explicación
numeroPrimo DS:0000 07 1 byte. 7 decimal = 07h
primoGrande DS:0001–0002 F5 26 2 bytes. 9973 = 26F5h. Little endian: byte bajo (F5h) primero, byte alto (26h) después
impares[0..3] DS:0003–0006 01 03 07 09 4 bytes consecutivos
arreglo_5[0..4] DS:0007–000B 0F 0F 0F 0F 0F 5 copias de 0Fh (DUP)
nombre[0..3] DS:000C–000F 4C 75 69 73 'L'=4Ch, 'u'=75h, 'i'=69h, 's'=73h (ASCII)
Little Endian es el orden en que el 8086 guarda los datos de 16 bits en memoria: el byte menos significativo (el "de la derecha" en hex) se guarda primero (en la dirección más baja). Así, 9973 = 26F5h se guarda como F5h en la primera celda y 26h en la segunda. Si los lees en orden de memoria, parecen "al revés".

Ejercicio 7 — Operaciones sobre las variables del Ej. 6

a) Copiar primoGrande en AX

mov ax, primoGrande ; AX = 9973 = 26F5h

Como AX es de 16 bits y primoGrande es DW (16 bits), los tamaños coinciden. El ensamblador carga los 2 bytes de primoGrande en AX automáticamente (deshaciendo el little endian).

b) Copiar el tercer elemento de impares (el número 7) en AL

impares DB 1, 3, 7, 9 → el elemento "número 7" está en el índice 2 (los índices arrancan en 0):

mov al, impares[2] ; AL = impares[2] = 7 (el tercer elemento)

c) Asignar BL, CL, DL a los primeros elementos de arreglo_5

mov arreglo_5, bl ; arreglo_5[0] = BL
mov arreglo_5[1], cl ; arreglo_5[1] = CL
mov arreglo_5[2], dl ; arreglo_5[2] = DL

c) Cambiar la variable nombre para que diga "Ruiz" en vez de "Luis"

"Luis" tiene 4 caracteres, "Ruiz" también tiene 4 → podemos sobreescribir byte a byte:

mov nombre, 'R' ; nombre[0] = 'R' = 52h
mov nombre[1], 'u' ; nombre[1] = 'u' = 75h
mov nombre[2], 'i' ; nombre[2] = 'i' = 69h
mov nombre[3], 'z' ; nombre[3] = 'z' = 7Ah

Memoria y Modos de Direccionamiento

Para acceder a una celda de memoria en Assembler usamos la notación DS:[dirección]. Lo que va entre corchetes es la dirección del dato.

ModoEjemploCómo se calcula la dirección
Inmediato mov ax, 50h El valor está en el propio código (constante)
De registro (Inherente) mov ax, bx El valor está en un registro
Directo mov ax, ds:[107Ah] La dirección está escrita directamente
Directo con variable mov ax, miVar La dirección la calcula el ensamblador a partir del nombre
Indirecto por registro mov ax, ds:[bx] La dirección está guardada en BX, BP, SI o DI
Relativo (base + índice) mov ax, arr[si] Dirección = base del arreglo + valor de SI
Relativo compuesto mov ax, ds:[bx+si+5] Dirección = BX + SI + 5
Registros válidos para direccionamiento indirecto: BX, BP, SI, DI. No podés usar AX, CX o DX entre corchetes para direccionar memoria.

Control de Flujo: CMP, JMP y LOOP

CMP — comparación

CMP hace una resta interna pero no guarda el resultado. Solo modifica el registro de flags. El salto condicional que viene después lee esos flags.

cmp ax, bx ; calcula AX - BX internamente, actualiza flags
je son_iguales ; si AX == BX (flag Z=1), salta

Saltos condicionales

InstrucciónSalta si...Flag que mira
JE / JZIgual a / es ceroZ = 1
JNE / JNZDistinto / no es ceroZ = 0
JA / JAEMayor / Mayor o igual (sin signo)C=0,Z=0 / C=0
JB / JBEMenor / Menor o igual (sin signo)C=1 / C=1 o Z=1
JG / JGEMayor / Mayor o igual (con signo)Z=0,S=O / S=O
JL / JLEMenor / Menor o igual (con signo)S≠O / S≠O o Z=1
JMPSiempre (incondicional)—

Estructura if/else en Assembler

; Python: if ax == bx: ... else: ...
cmp ax, bx ; comparar
jne bloque_else ; si AX≠BX → ir al else
; código del IF (AX == BX)
jmp fin_if
bloque_else:
; código del ELSE (AX ≠ BX)
fin_if:
; LOOP = for con CX como contador
mov cx, 10 ; 10 iteraciones
repetir:
; ... código a repetir ...
loop repetir ; CX--; si CX≠0 → saltar

; WHILE con condición
mientras:
cmp al, 0FFh
je salir ; condición de salida
; ... cuerpo del while ...
jmp mientras
salir:

Ejercicio 12 — Sumar 20 bytes de un arreglo

12
El arreglo empieza en la dirección que tiene BX. Guardar la suma en AL.
Estrategia: usar un loop con CX=20. En cada vuelta, accedemos a DS:[BX] (modo indirecto), sumamos a AL, incrementamos BX para apuntar al siguiente elemento.
ejercicio12.asm
.MODEL SMALL
.STACK
.DATA
datos DB 10,20,30,5,15,8,3,7,12,25,1,4,6,9,11,13,2,17,19,22
.CODE
inicio:
mov ax, @data
mov ds, ax

mov bx, offset datos ; BX = dirección base del arreglo
mov al, 0 ; AL = acumulador de la suma
mov cx, 20 ; CX = contador (20 elementos)

suma:
add al, ds:[bx] ; AL += valor del elemento actual
inc bx ; BX apunta al siguiente byte
loop suma ; CX--; repetir si CX≠0

; AL contiene la suma total
mov ax, 4C00h
int 21h
END inicio

Ejercicio 14

Ejercicio 14 — Copiar un arreglo (Numeros → Copia)

Estrategia: Usamos SI para indexar Numeros (fuente) y DI para indexar Copia (destino). En cada iteración copiamos un byte de uno al otro.
ejercicio14.asm
.DATA
Long DW 5 ; longitud del arreglo
Numeros DB 10, 20, 30, 40, 50 ; arreglo original
Copia DB 100 DUP(0) ; espacio para la copia
.CODE
inicio:
mov ax, @data
mov ds, ax

mov si, offset Numeros ; SI → inicio de la fuente
mov di, offset Copia ; DI → inicio del destino
mov cx, Long ; CX = cuántos elementos copiar

copiar:
mov al, ds:[si] ; leer byte de Numeros
mov ds:[di], al ; escribir byte en Copia
inc si ; avanzar en la fuente
inc di ; avanzar en el destino
loop copiar ; repetir Long veces

mov ax, 4C00h
int 21h
END inicio
Ejercicio 15

Ejercicio 15 — Longitud de nombre de archivo con extensión

Clave del problema: El nombre siempre termina en .XXX (punto + 3 letras = 4 caracteres). Buscamos el punto en el string, contamos cuántos caracteres hay antes, y le sumamos 4.
ejercicio15.asm
.DATA
NombreArch DB "Ejer1.ASM", 0 ; null terminado por seguridad
.CODE
inicio:
mov ax, @data
mov ds, ax

mov si, offset NombreArch ; SI apunta al inicio
mov cx, 0 ; CX = contador (longitud)

buscar:
mov al, ds:[si] ; leer caracter actual
cmp al, '.' ; ¿es el punto?
je encontrado ; sí → terminar la búsqueda
inc si ; no → avanzar al siguiente caracter
inc cx ; incrementar contador
jmp buscar

encontrado:
add cx, 4 ; sumar: punto(1) + 3 letras = 4 chars de extensión
; CX = longitud total del nombre con extensión

mov ax, 4C00h
int 21h
END inicio
Verificación: "Ejer1.ASM" → antes del punto: 'E','j','e','r','1' = 5 chars → CX=5 → +4 = 9 ✓  |  "Planilla.xls" → 8 chars antes del punto → +4 = 12 ✓
Ejercicio 16 — Avanzado

Ejercicio 16 — Extraer exponente de un número IEEE754

Contexto IEEE754 (precisión simple): Un número de punto flotante de 32 bits tiene: 1 bit de signo (bit 31), 8 bits de exponente (bits 30–23), 23 bits de mantisa (bits 22–0).

El número -0.000065 en IEEE754 es B888509Ch. En memoria (little endian con DD = double word): 9C 50 88 B8.

ejercicio16.asm — extracción del exponente e inversión del signo
.DATA
nroReal DD -0.000065 ; ocupa 4 bytes: 9C 50 88 B8
exponente DB ?
.CODE
inicio:
mov ax, @data
mov ds, ax

; Cargamos el byte alto de la parte alta del número (byte 3, el B8h)
; nroReal en mem: [9C][50][88][B8] → byte 2 = 88h (tiene bits del expo)
; En IEEE754: los bits 30-23 son el exponente
; byte2 = bits 23-16, byte3 = bits 31-24

mov al, nroReal[2] ; AL = 88h = byte 2 (bits 23-16)
mov ah, nroReal[3] ; AH = B8h = byte 3 (bits 31-24)
; AX = B888h. El exponente está en bits 30-23.
; Necesitamos aislar los 8 bits del exponente:
shr ax, 7 ; desplazar 7 bits a la derecha
and al, 0FFh ; aislar el byte bajo (el exponente)
mov exponente, al ; guardar exponente

; Invertir el signo: solo flipear el bit 31 (MSB del byte 3)
xor nroReal[3], 80h ; XOR con 80h invierte solo el bit 7 del byte 3

mov ax, 4C00h
int 21h
END inicio
Ejercicio 17

Ejercicio 17 — Convertir carácter ASCII a decimal

Clave: Los caracteres '0' a '9' tienen códigos ASCII consecutivos: '0'=30h, '1'=31h, ..., '9'=39h. Para convertir a número: restamos 30h. Para validar: verificamos que esté en ese rango.
ejercicio17.asm
.DATA
caracter DB '7' ; ASCII 37h = código de '7'
digito DB ?
.CODE
inicio:
mov ax, @data
mov ds, ax

mov al, caracter ; AL = código ASCII del caracter

cmp al, '0' ; ¿AL < '0' (30h)?
jb no_valido ; sí → no es un dígito
cmp al, '9' ; ¿AL > '9' (39h)?
ja no_valido ; sí → no es un dígito

sub al, '0' ; convertir: '7' - '0' = 37h - 30h = 7
mov digito, al ; guardar el dígito numérico
jmp fin

no_valido:
mov digito, 0FFh ; código de error

fin:
mov ax, 4C00h
int 21h
END inicio
Ejercicio 18

Ejercicio 18 — Invertir el orden de los bits de un byte

var = b7b6b5b4b3b2b1b0 → inv = b0b1b2b3b4b5b6b7

Algoritmo: En cada iteración: sacamos el bit menos significativo de AL con SHR (va al carry), y lo metemos al bit más significativo de BL con RCL (que rota hacia la izquierda metiendo el carry en el bit 0 y desplazando todo). Después de 8 vueltas, BL tiene los bits en orden invertido.
Traza con var = 11010010b:
AL: 11010010 BL: 00000000
SHR AL → carry=0 → AL=01101001
RCL BL → BL=00000000
... (8 iteraciones)
BL final: 01001011 ✓
.DATA
var DB 11010010b
inv DB ?
.CODE
inicio:
mov ax, @data
mov ds, ax
mov al, var
mov bl, 0 ; BL = resultado
mov cx, 8 ; 8 bits
invertir:
shr al, 1 ; LSB de AL → carry
rcl bl, 1 ; carry → entra al BL rotado
loop invertir
mov inv, bl
mov ax, 4C00h
int 21h
END inicio
Teoría

Subrutinas, la Pila y CALL/RET

Una subrutina en Assembler es como una función en Python o JavaScript. Se define con PROC/ENDP y se llama con CALL. Al terminar, usa RET para volver.

; Llamar a una subrutina
call miSubrutina

; Definir una subrutina
miSubrutina proc
; ... código ...
ret ; volver al punto de CALL
miSubrutina endp
; PUSH: apilar (guardar en la pila)
push ax ; SP -= 2; SS:[SP] = AX

; POP: desapilar (recuperar de la pila)
pop ax ; AX = SS:[SP]; SP += 2

; Preservar registros en subrutina
sub proc
push bx ; guardar
push cx ; guardar
; ... uso de BX y CX ...
pop cx ; restaurar (orden inverso!)
pop bx ; restaurar
ret
sub endp

La Pila (Stack) — cómo funciona

La pila es una estructura LIFO (último en entrar, primero en salir). SP siempre apunta al tope. Cuando hacés PUSH, SP baja (hacia direcciones menores). Con POP, SP sube.

Estado de la pila después de: push ax (AX=1234h), push bx (BX=5678h), call Subrutina

SP →
5678h
← valor de BX (lo último en entrar)
SP+2
dirección retorno
← puesto por CALL automáticamente
SP+4
1234h
← valor de AX (lo primero en entrar)

Convención con BP para parámetros

Para pasar parámetros a una subrutina vía la pila, se usa esta convención:

sub proc
push bp ; 1. guardar BP
mov bp, sp ; 2. BP = tope de pila
mov bx, ss:[bp+4] ; 3. 1er parámetro
mov cx, ss:[bp+6] ; 4. 2do parámetro
; ...
pop bp ; 5. restaurar BP
ret 4 ; 6. ret y limpiar N bytes de pila
sub endp
¿Por qué BP+4 y no BP+2? Después de hacer push bp y mov bp, sp: en [BP+0] está el BP original que guardamos, en [BP+2] está la dirección de retorno (puesta por CALL), y recién en [BP+4] empieza el primer parámetro que pusimos antes de llamar.

Ejercicio 24

Ejercicio 24 — Contar bits 1 (contarUnosByte / contarUnosWord)

contarUnosByte: cuenta los bits 1 en AL, devuelve en CX
contarUnosByte proc
push ax ; preservar AX
push dx ; usar DX como contador interno
mov cx, 0 ; CX = contador de 1s
mov dx, 8 ; DX = 8 iteraciones
cb_loop:
shr al, 1 ; sacar bit menos significativo → carry
jnc cb_no1 ; si carry = 0, el bit era 0
inc cx ; si carry = 1, el bit era 1 → incrementar
cb_no1:
dec dx
jnz cb_loop
pop dx
pop ax
ret
contarUnosByte endp
contarUnosWord: invoca dos veces a contarUnosByte
contarUnosWord proc ; AX = word, devuelve CX = total bits 1
push dx
call contarUnosByte ; contar bits en AL (parte baja)
mov dx, cx ; guardar conteo de la parte baja
push ax ; guardar AX
mov al, ah ; mover parte alta → AL
call contarUnosByte ; contar bits en AH (ahora en AL)
pop ax ; restaurar AX
add cx, dx ; CX = conteo total
pop dx
ret
contarUnosWord endp

Ejercicios 27–29 — Subrutinas con Strings

Ej 27: longString — longitud de string terminado en 0
longString proc ; recibe dirección del string en la pila, devuelve longitud en AX
push bp
mov bp, sp
mov si, ss:[bp+4] ; SI = dirección del string
mov ax, 0 ; AX = contador
ls_loop:
cmp byte ptr ds:[si], 0 ; ¿es el terminador nulo?
je ls_fin
inc si ; avanzar
inc ax ; contar
jmp ls_loop
ls_fin:
pop bp
ret 2 ; limpia los 2 bytes del parámetro (1 word)
longString endp

; Cómo llamarla:
mov ax, offset miString
push ax
call longString ; AX = longitud al retornar

Errores frecuentes

  • ❌ mov mem1, mem2 — no existe memoria a memoria
  • ❌ mov 5, ax — la constante no puede ser el destino
  • ❌ add ax, bl — tamaños distintos (16 vs 8 bits)
  • ❌ mov ds, 1234h — DS no acepta constantes directas
  • ❌ shl bh, 3 — el 2do operando de SHL debe ser 1 o CL
  • ✅ mov cl, 3 / shl bh, cl — correcto
  • ❌ Olvidar inicializar DS con mov ax, @data / mov ds, ax
  • ❌ En arreglos DW, usar índice 1 para el 2do elemento (debe ser 2)
