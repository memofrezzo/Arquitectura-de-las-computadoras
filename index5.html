Guía de Referencia Completa

Todo lo que necesitás entender antes de resolver los ejercicios

Cómo usar esta guía

Esta sección explica cada registro, cada instrucción y cada directiva que aparece en los ejercicios. Si algo te aparece en el código y no lo entendés, buscálo acá. Al final hay una tabla resumen con todo junto.

Los Registros — ¿Qué son?

Un registro es un lugar de almacenamiento dentro del propio procesador. Son como "variables ultra-rápidas" que la CPU usa directamente. Hay varios tipos:

Son los que más vas a usar. Cada uno tiene 16 bits y se puede dividir en dos mitades de 8 bits:

DXDatosDHDLLlamadas al DOS (DL=carácter a imprimir), datos temporales

Registros de Segmento

La memoria del 8086 se divide en segmentos de hasta 64KB. Cada registro de segmento "apunta" al inicio de un área:

¿Por qué "mov ds, @data"?

@data es un valor especial que el ensamblador reemplaza por la dirección donde empiezan tus variables. La instrucción mov ds, ax (con @data previamente en AX) hace que DS apunte a tus datos. Sin esto, cuando intentés leer una variable, el procesador buscaría en el lugar equivocado.

Registros de Pila: SP y BP

Estos dos son fundamentales para las subrutinas:

SP — Stack Pointer (Puntero de Pila)

SP siempre apunta al último dato que se apiló (el "tope" de la pila). Funciona así:

  • Cuando hacés PUSH, SP baja 2 (la pila crece hacia abajo)
  • Cuando hacés POP, SP sube 2 (la pila se encoge)
  • No se modifica directamente con MOV; se modifica con PUSH, POP, CALL, RET

Ejemplo: Si SS=0710h y la pila está vacía con SP=0100h:

  • Después de push ax: SP = 00FEh (bajó 2)
  • Después de otro push bx: SP = 00FCh (bajó 2 más)
BP — Base Pointer (Puntero Base)

BP se usa como punto de referencia fijo dentro de una subrutina para acceder a los parámetros que se pasaron por la pila.

¿Por qué es necesario? Porque SP cambia cada vez que hacés PUSH o POP dentro de la subrutina, así que no podés usar SP para encontrar los parámetros (se "mueve"). BP se fija una vez y queda quieto.

El patrón clásico es:

push bp     ; 1. Guardamos el BP viejo en la pila
mov bp, sp  ; 2. BP = SP actual. Ahora BP es nuestra "referencia"
             ;    todo parámetro queda en BP+4, BP+6, etc.

Registros Especiales: IP y FLAGS

IP — Instruction Pointer

IP guarda la dirección de la siguiente instrucción que la CPU va a ejecutar. La CPU lo incrementa automáticamente después de cada instrucción. Cuando se ejecuta CALL, la CPU guarda IP en la pila y lo cambia a la dirección de la subrutina. Con RET, lo recupera de la pila y vuelve a donde estaba.

FLAGS — Registro de Banderas

FLAGS es un registro especial que la CPU actualiza automáticamente después de cada operación aritmética o lógica. Las banderas más importantes son:

  • ZF (Zero Flag) = 1 si el resultado fue cero
  • CF (Carry Flag) = 1 si hubo acarreo (préstamo en resta)
  • SF (Sign Flag) = 1 si el resultado es negativo (bit más significativo = 1)
  • OF (Overflow Flag) = 1 si hubo desbordamiento con signo

Las instrucciones CMP y los saltos condicionales (JE, JB, JL...) leen estas banderas.

Segmentos y Memoria

Direcciones de memoria: Segmento:Offset

En el 8086, cada posición de memoria se identifica con dos números:

Segmento : Offset   →   Ejemplo: 0720:0003

Esto funciona como un "mes:día" en un calendario. El segmento selecciona un bloque de memoria y el offset indica la posición dentro de ese bloque.

Las variables en memoria

Cuando declarás variables en .DATA, el ensamblador las pone una detrás de otra en el segmento de datos. Ejemplo:

.data
  var1 DB 11         ; offset 0000 — 1 byte con valor 0Bh
  var2 DB 43h        ; offset 0001 — 1 byte con valor 43h
  var3 DB "ABC"      ; offset 0002 — 3 bytes: 41h, 42h, 43h

offset vs contenido

Cuando escribís mov ax, var2, leés el contenido de var2 (AX = 0043h).

Cuando escribís mov ax, offset var2, obtenés la dirección de var2 (AX = 0001h).

La diferencia es como preguntar "¿qué hay en la casa?" vs "¿cuál es la dirección de la casa?".

Little Endian (orden de bytes)

Cuando un dato ocupa 2 bytes (word), en memoria se guarda "al revés": el byte bajo primero y el byte alto después.

ValorDirección menorDirección mayor¿Por qué?
0035h35h (bajo)00h (alto)Convención del 8086
0720h20h (bajo)07h (alto)Siempre el byte bajo va primero

La Pila en Detalle — Paso a Paso

La pila es una zona de memoria que funciona como una torre de platos: el último que ponés es el primero que sacás (LIFO).

Cómo funciona PUSH (apilar)

  1. SP baja 2 (la pila crece hacia direcciones más bajas)
  2. El valor se escribe en la posición SS:SP

Ejemplo con PUSH AX (AX = 0033h, SP = 0100h):

Antes:  SP = 0100h
push ax
Paso 1: SP = SP - 2 = 00FEh
Paso 2: SS:[00FEh] = 0033h  (el valor de AX)
Después: SP = 00FEh

Cómo funciona POP (desapilar)

  1. El valor se lee de SS:SP
  2. SP sube 2

Cómo funciona CALL (llamar subrutina)

  1. SP baja 2
  2. Se guarda IP (la dirección de retorno) en SS:SP
  3. IP cambia a la dirección de la subrutina

Es como poner un "marcapáginas" en un libro antes de saltar a otro capítulo.

Cómo funciona RET (volver)

  1. Se lee IP de SS:SP
  2. SP sube 2
  3. La ejecución continúa donde quedó

Cómo funciona RET N (volver y limpiar parámetros)

Hace lo mismo que RET, pero además suma N a SP. Esto "borra" los parámetros que se habían pusheado antes del CALL.

ret 4  →  saca retorno + suma 4 a SP = limpia 2 parámetros

La estructura del "marco de pila" (Stack Frame)

Después de push bp y mov bp, sp dentro de la subrutina, la pila queda así:

PosiciónContenidoQuién lo puso
SS:[BP + 6]1er parámetro (el primer push)Tu código (antes del CALL)
SS:[BP + 4]Último parámetro (el último push)Tu código (antes del CALL)
SS:[BP + 2]Dirección de retorno (IP)La instrucción CALL
SS:[BP + 0]BP anteriorTu código (push bp)
¿Por qué BP+4 y no BP+0?

Porque las posiciones BP+0 y BP+2 están ocupadas por el BP guardado y la dirección de retorno. Los parámetros empiezan en BP+4. Si pusheaste más parámetros antes, el más viejo queda en BP+6, BP+8, etc. (en orden inverso a como los pusheaste).

Cada Instrucción Explicada

MOV — Mover (copiar) un valor
mov destino, fuente    ; destino = fuente

Copia el valor de la fuente al destino. No borra la fuente. Es como = en C. El destino no puede ser un número.

mov ax, 5        ; AX = 5
mov bx, ax       ; BX = AX = 5 (AX sigue valiendo 5)
PUSH — Apilar un valor en la pila
push valor   ; SP -= 2, luego guarda 'valor' en SS:[SP]

Pone un valor "arriba" de la pila. SP baja 2 automáticamente. El valor puede ser un registro o una variable (word).

POP — Desapilar (sacar de la pila)
pop destino   ; Lee SS:[SP], guarda en 'destino', luego SP += 2

Saca el último valor apilado. SP sube 2. Es el reverso de PUSH.

CALL — Llamar a una subrutina
call etiqueta   ; Guarda IP en la pila, salta a 'etiqueta'

Apila la dirección de retorno (IP) y salta a la subrutina. Es como un "GOTO con marcapáginas".

RET / RET N — Volver de una subrutina
ret       ; Saca IP de la pila, vuelve al llamador
ret 4    ; Hace ret + suma 4 a SP (limpia parámetros)

ret N limpia N bytes de parámetros. Si pusheados 2 parámetros (4 bytes) antes del CALL, usás ret 4.

CMP — Comparar (sin guardar resultado)
cmp op1, op2   ; Calcula op1 - op2 internamente, solo actualiza FLAGS

Resta op2 de op1 sin guardar el resultado. Solo modifica las banderas (ZF, CF, SF, OF). Se usa siempre seguido de un salto condicional.

Saltos Condicionales — Después de CMP

Estos saltos leen las banderas que dejó CMP:

InstrucciónSignificadoCondiciónTipo
JE / JZJump if EqualZF = 1 (op1 == op2)Ambos
JNE / JNZJump if Not EqualZF = 0 (op1 != op2)Ambos
JB / JNAEJump if BelowCF = 1 (op1 < op2 sin signo)Sin signo
JA / JNBEJump if AboveCF = 0 y ZF = 0 (op1 > op2 sin signo)Sin signo
JBEJump if Below or EqualCF = 1 o ZF = 1Sin signo
JAEJump if Above or EqualCF = 0Sin signo
JL / JNGEJump if LessSF ≠ OF (op1 < op2 con signo)Con signo
JG / JNLEJump if GreaterSF = OF y ZF = 0Con signo
¡Cuidado! JB vs JL

JB (Jump if Below) se usa para comparar números sin signo (0 a 255 o 0 a 65535). JL (Jump if Less) se usa para comparar números con signo (-128 a 127). El Ejercicio 1 usa JB y JA porque compara caracteres ASCII (valores sin signo). El Ejercicio 3 usa JL porque la tabla usada en clase asume comparación con signo.

LOOP — Repetir usando CX como contador
etiqueta:
    ; ... cuerpo del bucle ...
    loop etiqueta   ; CX = CX - 1; si CX ≠ 0, saltar a etiqueta

LOOP es como un for con CX como contador. Primero decrementa CX en 1, y si CX no llegó a cero, salta a la etiqueta. Si CX es cero, continúa con la siguiente instrucción.

Importante: CX debe cargarse con el número de iteraciones antes del bucle.

ADD y SUB — Sumar y restar
add destino, op2   ; destino = destino + op2
sub destino, op2   ; destino = destino - op2

El resultado siempre se guarda en el primer operando. El segundo operando puede ser un registro o un número.

INC y DEC — Incrementar y decrementar en 1
inc destino   ; destino = destino + 1
dec destino   ; destino = destino - 1

Equivalentes a ADD destino, 1 y SUB destino, 1, pero más rápidos.

INT 21h — Llamar al sistema operativo (DOS)

Ejecuta un servicio del DOS. El número de servicio va en AH. Los más usados:

AHQué haceParámetrosResultado

02hEscribir un carácter en pantallaDL = carácter—
09hEscribir un string en pantallaDX = dirección del string (hasta '$')—
4ChTerminar el programaAL = código de salida—

Ejercicio 1 — Subrutina de Conversión ASCII

Convertir un carácter de dígito a su valor numérico usando la pila

Código del programa
stack
.data
  dato1      db 3          ; 1 byte con valor 3
  dato2      dw "3"        ; 1 word con ASCII de '3' = 0033h
  resultado  db ?          ; 1 byte sin inicializar
.code
inicio:
  mov ax, @data            ; AX = dirección del segmento de datos
  mov ds, ax               ; DS = AX. Ahora DS apunta a nuestros datos
  mov ax, dato2            ; AX = contenido de dato2 = 0033h
  push ax                  ; Apila 0033h en la pila (parámetro)
  call subrutina           ; Llama a la subrutina
  mov resultado, ah        ; Guarda AH (devuelto por la subrutina) en resultado
  mov ax, 4C00h           ; Función: terminar programa
  int 21h                  ; Llama al DOS para terminar

subrutina proc
  push bp                  ; Guarda el BP anterior en la pila
  mov bp, sp               ; BP = SP. Ahora BP es nuestra referencia
  mov bx, ss:[bp+4]      ; ← (Ref.nro1) Lee parámetro de la pila → BX
  cmp bl, "0"             ; ¿BL < '0'? (compara sin signo)
  jb label1               ; Si es menor que '0', no es dígito → ir a label1
  cmp bl, "9"             ; ¿BL > '9'?
  ja label1               ; Si es mayor que '9', no es dígito → ir a label1
  sub bl, "0"             ; BL = BL - 30h. Convierte ASCII → número
  mov ah, bl              ; AH = resultado (para devolver al llamador)
  pop bp                   ; Restaura el BP anterior
  ret 2                    ; Vuelve + limpia 2 bytes (1 parámetro)
label1:
  mov ah, "#"             ; No era dígito → AH = '#'
  pop bp                   ; Restaura BP
  ret 2                    ; Vuelve + limpia
subrutina endp
end inicio
¿Qué hace este programa? (En criollo)

El programa tiene una "función" (subrutina) que recibe un valor por la pila. Si el byte bajo de ese valor es un carácter entre '0' y '9', lo convierte a número (le resta '0'). Si no, devuelve '#'. Es como hacer: función('3') → devuelve 3.

a) Estado de BX luego de mov bx, ss:[bp+4]

Seguimos cada instrucción hasta llegar al punto marcado

1. mov ax, dato2 — Lee la variable dato2. Como es DW (word), el ensamblador lee 2 bytes a partir del offset 0001h:

  • Byte bajo (offset 0001h) = 33h
  • Byte alto (offset 0002h) = 00h

AX = 0033h. (AH=00h, AL=33h)

2. push ax — Apila AX=0033h. SP baja de 0100h a 00FEh.

3. call subrutina — Apila la dirección de retorno. SP baja a 00FCh.

4. push bp (dentro de subrutina) — Apila el BP viejo. SP baja a 00FAh.

5. mov bp, sp — BP = 00FAh.

6. mov bx, ss:[bp+4] — Lee de la pila la posición BP+4:

BP + 4 = 00FAh + 4 = 00FEh

En SS:00FEh está el valor que pusheamos con push ax: 0033h.

BX = SS:[00FEh] = 0033h
BX

BH = 00h   BL = 33h
En palabras

BX vale 0033h. BL (el byte bajo) es 33h = ASCII de '3'. BH (el byte alto) es 00h. La subrutina usará BL para verificar si '3' es un dígito válido (lo es, porque está entre '0'=30h y '9'=39h).

b) Dirección apuntada por SS:[BP+4]

Ya sabemos que después de mov bp, sp, BP = 00FAh.

BP + 4 = 00FAh + 4 = 00FEh

La dirección completa es SS:00FEh = 0710:00FEh.

En esa posición hay 0033h, que es el parámetro que pusheamos antes del CALL.

Recordatorio

ss:[bp+4] significa: "en el segmento de pila (SS), empezá en donde apunta BP y avanzá 4 bytes". Eso te lleva al último parámetro que se pusheado. Ver la sección "La Pila en Detalle" si necesitás repasar.

Ejercicio 2 — Impresión y Limpieza de Pila

Subrutina que imprime un string y un carácter

Código del programa
.model small
.stack
.data
  var1  DB 11           ; 1 byte con valor 11 = 0Bh
  var2  DB 43h          ; 1 byte con valor 43h = ASCII 'C'
  var3  DB "Entrada:5"  ; 9 bytes con los caracteres del string
.code
inicio:
  mov ax, @data         ; AX = dirección del segmento de datos
  mov ds, ax            ; DS = AX
  push ax               ; 1er parámetro: el segmento de datos (0720h)
  mov dx, offset var2
  push dx               ; 2do parámetro: offset de var2 (0001h)
  call subrutina        ; Llama a la subrutina
  mov ax, 4C00h        ; Terminar
  int 21h

subrutina proc
  push bp               ; Guarda BP viejo en la pila
  mov bp, sp            ; BP = SP → referencia fija
  mov dx, offset var3  ; DX = dirección de var3
  mov ah, 09h           ; Función 09h: imprimir string
  int 21h               ; → Imprime "Entrada:5"
  mov bx, ss:[bp+4]   ; BX = 2do parámetro = offset de var2 = 0001h
  mov dl, ds:[bx]      ; DL = DS:[0001h] = 43h = 'C'
  mov ah, 02h           ; Función 02h: imprimir carácter
  int 21h               ; → Imprime 'C'
  pop bp                ; Restaura BP
  ret ?                 ; ← (Ref.nro2) ¿Cuánto limpiar?
subrutina endp
end inicio
¿Qué hace? (En criollo)

Pushea dos parámetros (ax y dx) y llama a una subrutina. La subrutina imprime un string ("Entrada:5") y luego lee un byte de la dirección pasada como parámetro e imprime ese byte como carácter ('C').

a) ¿Qué valor de RET asegura una correcta salida de pila?

Antes del call subrutina, se pushearon dos parámetros:

  1. push ax → 2 bytes (0720h)
  2. push dx → 2 bytes (0001h)

Total de bytes de parámetros: 4 bytes.

La fórmula es: ret N donde N = 2 × cantidad de parámetros.

N = 2 × 2 parámetros = 4   →   ret 4
¿Qué pasa si ponés ret ? o ret a secas?

Si no ponés el número, el RET solo saca la dirección de retorno de la pila, pero no limpia los 4 bytes de parámetros. Quedan como "basura" en la pila. Si la subrutina se llama muchas veces, la pila se va llenando de basura hasta desbordar.

Respuesta: ret 4

b) Salida por pantalla

Primera impresión: INT 21h con AH=09h
mov dx, offset var3   ; DX = 0002h (dirección de "Entrada:5")
mov ah, 09h           ; Función: imprimir string hasta encontrar '$'
int 21h               ; → Imprime: Entrada:5

La función 09h imprime desde la dirección en DX hasta encontrar un '$'. Si no hay '$', imprime indefinidamente (para este ejercicio asumimos que imprime "Entrada:5").

Segunda impresión: INT 21h con AH=02h
mov bx, ss:[bp+4]   ; BX = 0001h (el offset de var2 que pusheamos)
mov dl, ds:[bx]      ; DL = DS:[0001h] = 43h = 'C'
mov ah, 02h           ; Función: imprimir un carácter
int 21h               ; → Imprime: C

Primero se lee el offset de var2 de la pila (0001h). Luego se lee el byte en DS:0001h, que es 43h = 'C'. Finalmente se imprime ese carácter.

Ejercicio 3 — Buscar el Mayor en un Arreglo

Subrutina que recorre un arreglo de words y devuelve el mayor

Código del programa
.model small
.stack
.data
  arreglo  dw '5','6','1','4','8','7'  ; 6 words (12 bytes)
  str1     db 'El mayor es: $'          ; String para imprimir
  mayor    dw ?                        ; Word sin inicializar
  long     dw 6                        ; Longitud del arreglo
.code
inicio:
  mov ax, @data
  mov ds, ax
  push offset arreglo  ; 1er parámetro: dirección del arreglo
  push long             ; 2do parámetro: longitud (6)
  call mayorArreglo
  mov bx, ax           ; BX = resultado (AX devuelto por subrutina)
  mov mayor, bx        ; Guarda en variable 'mayor'
  mov dx, offset str1
  mov ah, 09h
  int 21h               ; Imprime "El mayor es: "
  mov dl, bl           ; DL = byte bajo de BX (= mayor)
  mov ah, 02h
  int 21h               ; Imprime el carácter del mayor
  mov ax, 4c00h
  int 21h

mayorArreglo proc
  push bp              ; Guarda BP viejo
  mov bp, sp           ; BP = SP (referencia fija)
  mov bx, ss:[bp+6]  ; BX = 1er parámetro = offset arreglo
  mov cx, ss:[bp+4]  ; CX = 2do parámetro = longitud (6)
  mov ax, 0           ; AX = 0 (mayor actual)
iterar:
  cmp ds:[bx], ax     ; ¿elemento_actual < AX?
  jl menor              ; Si es menor (con signo), salta
  mov ax, ds:[bx]     ; ← X: Actualiza AX con el nuevo mayor
menor:
  add bx, 2           ; Avanza al siguiente word (+2 bytes)
  loop iterar          ; CX--; si CX≠0 vuelve a iterar
  pop bp               ; Restaura BP
  ret 4                ; Vuelve + limpia 2 parámetros (4 bytes)
mayorArreglo endp
END inicio
¿Qué hace? (En criollo)

Recibe la dirección de un arreglo y su longitud. Recorre el arreglo comparando cada elemento con el mayor encontrado hasta ahora (empezando de 0). Devuelve el mayor en AX. Es como int mayor = 0; for(i=0;i<6;i++) if(arr[i]>mayor) mayor=arr[i];

Valores del arreglo (son caracteres ASCII como words):
Pos.CharASCIIWordLittle Endian
[0]'5'35h = 530035h35h, 00h
[1]'6'36h = 540036h36h, 00h
[2]'1'31h = 490031h31h, 00h
[3]'4'34h = 520034h34h, 00h
[4]'8'38h = 560038h38h, 00h
[5]'7'37h = 550037h37h, 00h

El programa busca el mayor de [53, 54, 49, 52, 56, 55]. El mayor es 56 = 0038h = '8'.

[2 pts] Salida por pantalla

Después de la subrutina, AX = 0038h. El programa hace:

mov bx, ax           ; BX = 0038h
mov mayor, bx        ; Guarda 0038h en 'mayor'
mov dx, offset str1  ; DX = dirección de "El mayor es: $"
mov ah, 09h
int 21h               ; → Imprime "El mayor es: "
mov dl, bl           ; DL = 38h = ASCII '8'
mov ah, 02h
int 21h               ; → Imprime '8'
El mayor es: 8
¿Por qué imprime '8' y no 56?

INT 21h / AH=02h imprime el carácter cuyo ASCII está en DL. DL = 38h = '8'. El arreglo tiene los ASCII de los dígitos, así que el mayor ASCII (38h = '8') se imprime directamente como el dígito '8'.

[1 pt] ¿Qué valor toma AX en la posición marcada (X) luego de la 3ra iteración?

Análisis de la 3ra iteración
  • BX = 0004h → apunta a arreglo[2] = 0031h (=49 = '1')
  • AX = 0036h (=54 = '6') ← era el mayor hasta ahora
  • cmp ds:[bx], ax → compara 49 con 54 (resta internamente: 49 - 54 = -5)
  • Como 49 < 54, jl menor sí salta
  • No llega a la línea X, AX no se modifica
AX = 0036h  (= 54 = ASCII '6')
En palabras

El '1' (31h=49) es más chico que el mayor actual '6' (36h=54), así que el salto jl menor se activa y se saltea la línea de actualización. AX queda con 0036h.
